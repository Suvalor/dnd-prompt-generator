# DND 独立指南内容规划

## 核心判断

“真正独立”不是为不同关键词复制同一篇文章，而是每篇解决一个不同的
创作任务，并提供该任务独有的判断标准、失败案例和修改方法。

第一阶段建议写 6 篇核心指南，对应生成器现有的六种输出类型。第二阶段再写
2 篇跨类型的模型适配与故障排查指南。

Google 没有公布固定的最低文章数或字数。这里的数量是本站内容架构建议，
不是 AdSense 审核门槛。

## 目标读者

- 正在为角色制作头像或全身立绘的 DND 玩家。
- 需要快速制作 NPC、怪物和场景素材的 DM。
- 使用 Foundry VTT、Roll20 等工具，需要可读 Token 的用户。
- 不知道如何把同一个角色描述改写给不同图像模型的用户。
- 已经生成图片，但遇到构图、裁切、比例或主体混乱问题的用户。

## 第一批优先发布

建议先完成以下 3 篇。它们的使用场景和视觉规则差异最大，最容易证明内容
不是批量模板。

### 1. Top-Down DND Token Prompt Guide

**建议 URL**

```text
/guides/dnd-token-prompt-guide
```

**核心问题**

如何生成缩小到 VTT 棋盘尺寸后仍然清晰、居中、可识别的俯视角色 Token？

**读者读完应得到**

- 判断一个 Token 是否合格的检查标准。
- 正确表达俯视角、居中、轮廓和背景的方法。
- 将普通角色立绘提示词改成 Token 提示词的步骤。

**独有内容**

1. 俯视角与普通正面肖像的区别。
2. 为什么必须控制轮廓、道具数量和地面阴影。
3. 圆形裁切、透明背景和 `1:1` 画幅如何配合。
4. 在 512×512 和棋盘缩略尺寸下检查可读性。
5. 常见失败：
   - 变成正面人物照。
   - 出现复杂场景。
   - 武器或披风被圆形边界裁断。
   - 出现多个角色。
6. 三组“失败提示词 → 修改过程 → 最终提示词”。
7. Midjourney、ChatGPT 和 Stable Diffusion 的 Token 表达差异。
8. 导出到 VTT 前的检查清单。

**必须准备的原创证据**

- 至少 3 个不同体型或职业的实际测试案例。
- 每个案例记录失败原因，而不只是展示最终提示词。
- 一张 Token 缩小后的可读性对比图。

### 2. DND Character Portrait Prompt Guide

**建议 URL**

```text
/guides/dnd-character-portrait-prompt-guide
```

**核心问题**

如何让 AI 头像同时表达种族、职业、性格和光线，而不是生成普通奇幻人物？

**读者读完应得到**

- 知道哪些身份信息必须进入提示词。
- 学会控制脸部、半身构图、视线和背景。
- 能减少服装、武器和背景对面部的干扰。

**独有内容**

1. 角色身份的四层结构：
   - 种族视觉特征。
   - 职业识别元素。
   - 性格或情绪。
   - 一项记忆点。
2. Headshot、head-and-shoulders、bust portrait 的差异。
3. 三分之四视角、眼平视角和低角度的选择。
4. 主光、轮廓光和背景光如何影响角色气质。
5. 常见失败：
   - 角色看起来像通用奇幻模特。
   - 武器遮挡面部。
   - 画面变成全身照。
   - 种族特征过度或缺失。
6. Tiefling Warlock、Elf Ranger、Dragonborn Paladin 三个完整案例。
7. 如何把角色背景故事压缩为一个可见细节。
8. 肖像提示词检查清单。

**必须准备的原创证据**

- 同一角色在删除和加入“记忆点”前后的比较。
- 至少一例解释为什么删掉某些设定反而更好。
- 对生成器当前 Portrait 规则的人工评审。

### 3. DND Monster Prompt Guide

**建议 URL**

```text
/guides/dnd-monster-prompt-guide
```

**核心问题**

如何让怪物的解剖、尺寸、威胁方式和栖息环境形成统一设计？

**读者读完应得到**

- 能从“怪物名称”扩展出可视化的身体结构。
- 知道如何表达体型和比例。
- 能避免怪物变成普通动物、可爱生物或无尺度概念图。

**独有内容**

1. 怪物设计的五个字段：
   - 基础身体结构。
   - 异常部位。
   - 移动方式。
   - 攻击或威胁方式。
   - 栖息环境。
2. 使用人类、门、树木或地平线作为尺度参照。
3. 低机位、广角和环境构图如何表现体型。
4. 保持混合生物解剖一致的方法。
5. 常见失败：
   - 多出肢体但没有结构逻辑。
   - 怪物没有尺寸感。
   - 环境喧宾夺主。
   - 恐怖怪物变成卡通宠物。
6. 小型伏击怪、中型首领和巨型怪物三个完整案例。
7. 负面提示词什么时候有效，什么时候会制造新问题。
8. 怪物设计检查清单。

**必须准备的原创证据**

- 三种尺寸级别的对比。
- 一例混合生物解剖失败与修正。
- 解释尺度参照为何改变最终画面。

## 第二批核心指南

### 4. DND Full-Body Character Prompt Guide

**建议 URL**

```text
/guides/dnd-full-body-character-prompt-guide
```

**核心问题**

如何在完整展示盔甲、武器和姿势的同时，避免四肢被裁切或装备堆叠混乱？

**与肖像指南的独立边界**

- 肖像指南关注脸部、身份和光线。
- 本篇关注完整轮廓、动作重心、装备层级和画幅。

**独有章节**

- Full-body、full-length、character sheet 的区别。
- 站立、行走、攻击和防御姿势的动词写法。
- 主武器与次要装备的优先级。
- `2:3` 画幅与脚部留白。
- 裁切、手部、武器融合、悬浮姿势的修正案例。
- 同一角色从肖像提示词转换成全身提示词的过程。

### 5. Memorable DND NPC Prompt Guide

**建议 URL**

```text
/guides/dnd-npc-prompt-guide
```

**核心问题**

如何用职业、社会角色和一个记忆点，让 NPC 一眼可辨，而不是生成普通人物？

**独有章节**

- “剧情职业”与“战斗职业”的区别。
- 用服装磨损、工具和表情表达 NPC 的日常生活。
- 一项记忆点原则：金牙、账本、旧徽章、染色手套等。
- 商人、守卫、酒馆老板、贵族顾问的角色视觉语言。
- 如何生成同一阵营的一组 NPC，同时保持个体区别。
- 避免现代服装、摄影棚肖像和通用脸。
- 一个 NPC 从文字设定到可见细节的完整转换。

### 6. Fantasy Scene and Encounter Prompt Guide

**建议 URL**

```text
/guides/fantasy-scene-encounter-prompt-guide
```

**核心问题**

如何设计有空间层次、时间、天气和遭遇线索的场景，而不是只有一张空泛背景图？

**独有章节**

- 前景、中景、背景的三层结构。
- 地点、时间、天气、光源、事件痕迹五项信息。
- Establishing shot、battle map mood art、location concept 的区别。
- 如何加入遭遇线索，但不让中央英雄抢走场景主体。
- 酒馆、遗迹、森林伏击点三个案例。
- 常见失败：空间扁平、尺度不明、光源矛盾、人物喧宾夺主。
- `16:9` 和其他场景画幅的选择。

## 第三批跨类型指南

### 7. Midjourney vs ChatGPT vs Stable Diffusion for DND Prompts

**建议 URL**

```text
/guides/dnd-prompt-model-comparison
```

**核心问题**

同一个角色或场景，为什么不能把完全相同的提示词原样复制给所有模型？

**独有内容**

- 三个平台的提示词组织差异。
- Midjourney 参数、自然语言指令和 Stable Diffusion 权重的区别。
- 同一角色的三种改写示例。
- 负面提示词支持差异。
- 何时应该缩短提示词。
- 选择模型的任务决策表。

不要写成模型功能百科。重点必须是 DND 视觉任务中的实际差异。

### 8. How to Fix Common DND AI Art Prompt Failures

**建议 URL**

```text
/guides/fix-dnd-ai-art-prompt-failures
```

**核心问题**

图片结果不对时，应该修改哪一段提示词，而不是不断增加形容词？

**独有内容**

- 主体错误。
- 构图和裁切错误。
- 多角色错误。
- 武器和手部错误。
- Token 角度错误。
- 背景过度复杂。
- 风格和写实程度漂移。
- 每类问题的“症状 → 原因 → 最小修改”表格。
- 什么时候重新生成，什么时候局部修改。

这篇应作为所有核心指南的故障排查中心，并从其他指南链接过来。

## 每篇统一框架

结构可以一致，但内容不能只是替换关键词。

1. **问题场景**
   - 用户正在做什么。
   - 最常见的失败是什么。
2. **成功标准**
   - 给出该类型独有的 4–6 条判断标准。
3. **提示词构造方法**
   - 按决策顺序拆解，不只是罗列字段。
4. **完整案例**
   - 原始需求。
   - 第一版提示词。
   - 失败结果或风险。
   - 修改理由。
   - 最终提示词。
5. **常见错误**
   - 症状、原因和最小修正。
6. **模型差异**
   - 只写与本任务相关的差异。
7. **检查清单**
   - 用户复制提示词前可以逐项核对。
8. **进入生成器**
   - 使用预填参数打开对应生成器类型。

## 独立性验收标准

发布前逐篇检查：

- 这篇是否解决一个其他页面不能替代的问题？
- 是否有该类型独有的成功标准？
- 是否包含至少 3 个经过人工解释的案例？
- 是否展示修改原因，而不只是最终答案？
- 是否包含真实失败模式？
- 示例是否经过人工检查，而非批量自动生成？
- 标题、H1、描述、FAQ 是否与其他指南明显不同？
- 删除种族或职业名称后，文章是否仍然有独立价值？
- 用户不使用生成器，是否仍能从页面得到完整方法？

如果最后一个问题答案是否定的，这篇通常仍然只是工具落地页，不是独立指南。

## 不建议现在写的页面

暂时不要批量发布：

- Tiefling prompt generator。
- Elf prompt generator。
- Paladin prompt generator。
- 每个种族 × 每个职业的组合页。
- 只替换名称和示例的风格页。
- 未经人工测试的 AI 提示词合集。

这些页面容易重复，也无法证明相对其他网站具有实质原创价值。

## 与 AdSense 复审的关系

在申请复审前，至少应做到：

- 核心指南已经真实发布，而不是显示“即将上线”。
- 页面能通过站内导航访问。
- 每篇有独立 canonical 并进入 sitemap。
- 页面不是首页 SPA fallback。
- Contact、Privacy、Terms、404 等页面不展示广告。
- 指南内容成为页面主体，广告不靠近生成按钮、复制按钮或结果区域。
- Search Console 实时测试能够读取正确内容。

发布数量本身不能保证审核通过。审核重点仍然是内容是否原创、有用、完整，
以及广告是否只出现在包含发布商内容的页面。
